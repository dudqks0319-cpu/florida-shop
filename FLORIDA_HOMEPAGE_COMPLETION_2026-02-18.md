# FLORIDA 홈페이지 미완성 원인 진단 및 보완 결과 (2026-02-18)

## 왜 미완성이었는가

1. **카테고리/검색이 실데이터와 분리**
   - 홈 카테고리 아이콘은 단순 텍스트 버튼만 있고 실제 필터 동작이 없었음.
   - `/florida/all` 페이지가 정적 더미(원형 플레이스홀더) 중심이라 검색/카테고리 흐름이 끊겼음.

2. **핵심 탐색 흐름 단절**
   - SearchModal이 `?search=` 쿼리를 보내도 전체보기 페이지에서 반영하지 않았음.
   - 하단 네비게이션이 `찜`/`마이페이지` 모두 같은 경로를 가리켜 정보 구조가 부정확했음.

3. **프로모션/혜택 랜딩 정합성 부족**
   - 웰컴 페이지가 브랜드 문구(ABLY) 중심 더미 텍스트로 남아있어 FLORIDA 도메인 톤과 맞지 않았음.
   - 상단 "앱에서 보기" CTA가 실제 혜택 흐름과 약하게 연결되어 있었음.

4. **홈 실사용성 부족**
   - 추천 섹션이 고정 8개 노출 중심이라 사용자 의도(가격/리뷰/카테고리)에 따라 탐색하기 어려웠음.

---

## 이번 보완 내용

### 1) 홈(`/florida`)을 실사용 탐색형으로 개편
- 카테고리 선택, 검색어 입력, 정렬(추천/낮은가격/높은가격/리뷰순) 추가
- 결과 건수/빈결과 UX 추가
- "오늘의 추천 TOP 3" 카드 추가
- 타임딜 버튼 클릭 시 필터 자동 적용(영캐주얼 + 가격순)

### 2) 카테고리 바 컴포넌트 기능화
- `CategoryBar`를 단순 표시에서 **선택 가능한 필터 UI**로 변경
- 선택 상태 하이라이트 + 클릭 이벤트 제공

### 3) 전체보기(`/florida/all`) 실동작화
- 정적 더미 레이아웃 제거
- 실제 상품 데이터 기반 목록/필터/정렬/검색 연결
- 좌측 카테고리 선택 + 우측 상품 카드 실연동
- 찜/장바구니/최근본상품 로직 유지

### 4) 검색 모달 품질 개선
- 하드코딩 키워드 대신 실제 상품명/설명/카테고리 기반 추천어 제공
- 검색 결과를 `/florida/all?search=` 흐름으로 자연 연결

### 5) 웰컴 페이지 정합성 복구
- ABLY 텍스트 제거
- FLORIDA 브랜드 혜택 랜딩으로 재구성
- 로그인/둘러보기 CTA 동작 연결

### 6) 하단 네비게이션 정리
- 중복 경로(찜/마이페이지) 문제 해소
- 홈/카테고리/혜택/장바구니/마이로 단순 명확화

---

## 기대 효과

- 검색 → 카테고리 → 상품상세 → 장바구니로 이어지는 **탐색-구매 흐름 복구**
- 더미 화면에서 벗어나 **실제 운영 가능한 MVP 수준의 홈/카테고리 완성도 확보**
- 브랜드 정체성(FLORIDA)과 랜딩 문구 정합성 개선
