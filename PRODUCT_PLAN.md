# 동네 건당 심부름 앱 실행 계획 (MVP)

## 1) 서비스 포지셔닝
- 대상: 같은 아파트/동네 주민
- 문제: 짧은 생활 심부름(편의점, 배달대리수령, 은행/행정 대기)을 빠르게 맡길 사람 없음
- 해법: 건당 보상금으로 즉시 매칭하는 신뢰 기반 마이크로 심부름

## 2) 타 플랫폼 대비 차별점
- 당근: 동네 기반 강하나 ‘건당 심부름’ 전용 플로우 부족
- 숨고: 전문가 매칭 중심으로 즉시성 심부름과 거리감
- 우리의 초점: ‘15~30분 생활 심부름’ + ‘아파트 단지 인증’ + ‘패널티 규칙 명확화’

## 3) 수익모델
1. 정상 완료 건 중개수수료 8~12%
2. 급행 노출권(상단 고정) 유료
3. 안전결제/분쟁처리 수수료

## 4) 신뢰/패널티 규칙 (초안)
- 의뢰 등록 시 소액 보증금/예약금
- 의뢰자 위반(노쇼/무응답): 수행자 보상금 지급
- 수행자 위반(무단 미이행): 의뢰자 보상금 지급
- 반복 위반 계정 제한

## 5) MVP 화면
1. 의뢰 등록
2. 의뢰 목록/상세
3. 매칭/진행/완료 상태 관리
4. 기본 정책/신뢰 안내
5. 운영 요약 대시보드

## 6) 마케팅 실행 (파일럿)
- 범위: 아파트 단지 1곳에서 시작(2주)
- 채널: 입주민 카페/오픈채팅/엘리베이터 안내 QR
- 혜택: 첫 의뢰 수수료 무료 + 첫 수행 보너스
- KPI: 등록 50건, 완료율 80%+, 재사용률 30%+

## 7) 다음 개발 우선순위
1. 채팅/알림(매칭 즉시)
2. 분쟁 접수/증빙 업로드(중강도 패널티 이의제기)
3. 리뷰/신뢰점수(반복 위반 노출)
4. 관리자 페이지(정책 집행 + 제재)
5. 실제 결제 연동(가상 정산 → 실정산)

## 8) 반영 완료(2026-02-15)
- 동네 인증 완료 사용자만 의뢰 등록 가능하도록 API 가드 적용
- 파일럿 1개 단지 강제(첫 등록 단지 기준)
- 완료 시 자동 정산(플랫폼 10% / 수행자 90%)
- 상태 전이 검증(open→matched→in_progress→done/cancelled)
- 중강도 취소 패널티 자동 계산/기록 및 UI 표시
